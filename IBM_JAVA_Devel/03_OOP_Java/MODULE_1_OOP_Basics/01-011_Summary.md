# 01-011 Summary:   Object Oriented Programming Basics in Java

## Classes and Objects

- Classes serve as blueprints defining shared properties and methods for creating objects.

- Properties are an object's attributes, and methods define its actions.

- A class is a blueprint for creating objects that define attributes and methods.

- Objects are instances of classes created based on their class definition.

---

## Inheritance and Polymorphism

- Inheritance allows new classes to derive properties and methods from existing ones.

- Polymorphism enables objects to use the same interface while exhibiting different behaviors.

---

## Access Modifiers

- Access modifiers in Java control the visibility and accessibility of classes, methods, and variables.

- The four types of access modifiers are `public`, `private`, `protected`, and `default`.

---

## Non-Access Modifiers

- Non-access modifiers provide additional information about classes, methods, and variables, influencing their behavior without affecting visibility.

- The types of non-access modifiers are `static`, `final`, and `abstract`.

---

## Encapsulation

- Encapsulation ensures that an object's internal details are hidden, exposing only necessary information.

- Encapsulation bundles data and methods together, allowing easy updates to code without affecting other parts and enhancing security.

---

## Constructors

- Constructors are special methods to initialize Java objects.

- Java automatically provides default constructors if no constructors are defined.

- Parameterized constructors allow values to be passed during object creation, providing flexibility in initialization.

- No-arg constructors do not accept parameters and are used to create objects with default values or predefined values for its attributes.

- Best practices for using constructors include keeping them simple, using parameterized constructors for meaningful initialization, and focusing on initialization rather than complex logic.
